

# Android 平台针对 USB 功能的测试项

Android 平台针对 USB 功能的测试项，主要分为 **CTS（兼容性测试套件）**、**VTS（供应商测试套件）**、**手动功能验证** 三大类，覆盖 USB Host/Device/OTG、协议兼容性、设备功能、稳定性等核心场景。以下是详细的测试项分类与具体内容：

---

### 一、 核心测试框架：CTS + VTS
Android 官方的兼容性测试（CTS）和供应商测试（VTS）是 USB 功能合规性的核心依据，**必须通过才能获得 Google 认证**。
#### 1.  CTS 测试项（`android-cts/tests/tests/usb`）
面向 App 开发者与系统兼容性，验证 USB 相关 API 与基础功能。
| 测试模块           | 具体测试项                | 测试目的                                                 |
| ------------------ | ------------------------- | -------------------------------------------------------- |
| **USB Host API**   | `UsbHostManagerTest`      | 验证 `UsbManager` 枚举 USB 设备、获取设备信息的正确性    |
|                    | `UsbDeviceConnectionTest` | 测试 USB 设备连接/断开、数据传输（bulk/interrupt 端点）  |
| **USB Device API** | `UsbDeviceTest`           | 验证 Gadget 模式下设备描述符、接口描述符的正确性         |
|                    | `UsbAccessoryTest`        | 测试 USB Accessory 模式（外设作为主机，手机作为从机）    |
| **OTG 功能**       | `UsbOtgTest`              | 验证 OTG 模式自动切换（Host ↔ Device）、SRP/HNP 协议交互 |
| **权限与安全**     | `UsbPermissionTest`       | 测试 App 申请 USB 权限的流程、权限管控的有效性           |

#### 2.  VTS 测试项（`android-vts/testcases/usb`）
面向芯片/设备厂商，验证底层驱动、内核接口、硬件兼容性，**需 root 权限**。
| 测试模块             | 具体测试项             | 测试目的                                                     |
| -------------------- | ---------------------- | ------------------------------------------------------------ |
| **内核驱动验证**     | `VtsUsbXhciHcdTest`    | 测试 USB 3.0 XHCI 控制器驱动的稳定性、带宽性能               |
|                      | `VtsUsbGadgetTest`     | 验证 USB Gadget 框架（如 ACM/ECM/Mass Storage）的功能正确性  |
| **协议兼容性**       | `VtsUsbUvcTest`        | 测试 UVC 摄像头驱动、数据传输的稳定性（画面无卡顿/花屏）     |
|                      | `VtsUsbUacTest`        | 测试 UAC 音频设备的播放/录音功能、采样率兼容性               |
| **性能测试**         | `VtsUsbThroughputTest` | 测试 USB 传输吞吐量（读/写速度），验证是否达到 USB 2.0/3.0 标称带宽 |
| **Thunderbolt/USB4** | `VtsUsb4Test`          | 验证 USB4/Thunderbolt 协议协商、双角色切换、菊花链扩展       |

---

### 二、 手动功能测试项（研发/调试阶段）
在实验室或产线阶段，需手动验证 USB 全场景功能，覆盖 CTS/VTS 未包含的边缘场景。
#### 1.  USB Host 模式测试
| 测试项             | 操作步骤                                          | 验证标准                                                     |
| ------------------ | ------------------------------------------------- | ------------------------------------------------------------ |
| **外设枚举测试**   | 连接 U 盘、鼠标、键盘、UVC 摄像头、UAC 声卡等外设 | 系统能正确识别设备，生成 `/dev/block/sda`（U盘）、`/dev/video0`（摄像头）等节点 |
| **数据传输测试**   | 拷贝大文件（如 10GB）到 U 盘/从 U 盘拷贝          | 传输速度稳定，无丢包、文件损坏；USB 3.0 速度 ≥ 80MB/s        |
| **多外设并发测试** | 同时连接 U 盘 + 摄像头 + 声卡                     | 所有设备同时工作正常，无冲突、无掉线                         |
| **热插拔测试**     | 反复插拔 USB 外设（≥50 次）                       | 设备每次都能正常识别，无内核 panic、无系统卡死               |

#### 2.  USB Device 模式测试
| 测试项              | 操作步骤                                         | 验证标准                                                  |
| ------------------- | ------------------------------------------------ | --------------------------------------------------------- |
| **Gadget 功能测试** | 配置 ACM 串口、ECM 网卡、Mass Storage 等复合设备 | 主机能识别所有功能，串口可收发数据、网卡可联网、U盘可读写 |
| **USB 调试（ADB）** | 开启 USB 调试，连接 PC 执行 `adb shell`          | ADB 连接稳定，命令执行无卡顿                              |
| **MTP/PTP 测试**    | 切换到 MTP 模式，在 PC 上访问手机文件            | 文件可正常复制/删除，支持大文件（≥4GB）传输               |

#### 3.  OTG 模式测试
| 测试项               | 操作步骤                                                     | 验证标准                                                     |
| -------------------- | ------------------------------------------------------------ | ------------------------------------------------------------ |
| **角色切换测试**     | 连接 OTG 线 + U 盘（Host 模式），再连接另一台手机（Device 模式） | 系统自动切换角色，Host 模式可访问 U 盘，Device 模式可被 PC 识别 |
| **SRP 唤醒测试**     | 总线休眠后，从机发起 SRP 请求                                | 主机能被唤醒，重新建立 USB 会话                              |
| **HNP 角色互换测试** | 两台 OTG 设备连接，触发 HNP 切换                             | 角色成功互换，原从机变为主机，原主机变为从机                 |

#### 4.  特殊功能测试（Thunderbolt/USB4/FIDO2）
| 测试项                   | 操作步骤                            | 验证标准                                       |
| ------------------------ | ----------------------------------- | ---------------------------------------------- |
| **Thunderbolt 模式切换** | 连接雷电扩展坞、eGPU                | 系统识别为雷电设备，支持 PCIe 传输、菊花链扩展 |
| **USB4 带宽测试**        | 连接 USB4 移动硬盘，测试读写速度    | 速度达到 40Gbps 标称值（约 5000MB/s）          |
| **FIDO2 安全密钥测试**   | 连接 YubiKey，在 App 中完成身份认证 | 认证流程成功，日志无驱动报错                   |

---

### 三、 稳定性与压力测试项
用于验证 USB 功能在长期使用中的可靠性，通常在产线或可靠性测试阶段执行。
1.  **长时间压力测试**：连续 72 小时循环插拔 USB 外设 + 大文件传输，验证无内存泄漏、无驱动崩溃。
2.  **低功耗测试**：USB 休眠时测量功耗，验证 SRP 唤醒后功耗恢复正常，无异常耗电。
3.  **异常场景测试**：连接劣质线缆、供电不足的外设，验证系统不会卡死/重启，具备容错能力。

---

### 四、 测试工具与命令
Android 平台可通过以下工具快速执行 USB 测试：
| 工具/命令                             | 用途                  | 示例                                                   |
| ------------------------------------- | --------------------- | ------------------------------------------------------ |
| `adb shell lsusb`                     | 枚举 USB 设备         | `lsusb -t`（查看设备树）                               |
| `adb shell dmesg`                     | 查看 USB 驱动日志     | `dmesg | grep usb`                                     |
| `adb shell cat /proc/bus/usb/devices` | 查看 USB 设备描述符   | -                                                      |
| `tinycap/tinyplay`                    | 测试 UAC 音频功能     | `tinyplay test.wav -D 1`                               |
| `v4l2-ctl`                            | 测试 UVC 摄像头功能   | `v4l2-ctl --stream-mmap`                               |
| `cts-tradefed`                        | 运行 CTS USB 测试套件 | `cts-tradefed run singleCommand cts -m CtsUsbHostTest` |

---

### 总结
Android USB 测试的核心是 **CTS/VTS 合规测试 + 手动功能验证 + 稳定性压力测试**，覆盖从底层驱动到上层 API 的全链路。不同测试阶段（研发/认证/产线）需选择对应的测试项，确保 USB 功能在各种场景下稳定可靠。
