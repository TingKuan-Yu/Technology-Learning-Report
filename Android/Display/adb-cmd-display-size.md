# 如何用adb command來改變display resolution

你想要通过 ADB 命令来修改安卓设备的屏幕显示分辨率，这个需求很常见，尤其在调试或适配应用时非常实用。

### 操作方法详解
首先需要明确，使用 ADB 修改分辨率需要满足以下前提：
1. 你的安卓设备已开启 **开发者选项** 和 **USB 调试**
2. 电脑已安装 ADB 驱动并能正常连接设备
3. 部分设备（尤其是非原生安卓）可能需要 root 权限才能修改

#### 1. 查看当前分辨率
先执行以下命令查看设备当前的分辨率，方便后续恢复：
```bash
# 查看当前显示参数（包含分辨率）
adb shell wm size
# 示例输出：Physical size: 1080x2400（物理分辨率），Override size: 720x1600（当前修改后的分辨率）
```

#### 2. 修改分辨率
使用 `wm size` 命令修改分辨率，格式为 `宽度x高度`（注意是小写的 x）：
```bash
# 修改为 1080x1920 分辨率
adb shell wm size 1080x1920

# 修改为 720x1280 分辨率（720P）
adb shell wm size 720x1280

# 修改为 2560x1440 分辨率（2K）
adb shell wm size 2560x1440
```

#### 3. 恢复原始分辨率
如果想要恢复设备默认的物理分辨率，执行：
```bash
adb shell wm size reset
```

#### 4. 额外：修改屏幕密度（可选）
修改分辨率后，通常需要配合调整屏幕密度（DPI）来保证显示比例正常：
```bash
# 查看当前密度
adb shell wm density

# 修改密度（例如 480 DPI）
adb shell wm density 480

# 恢复默认密度
adb shell wm density reset
```

### 注意事项
- **分辨率兼容性**：请选择设备屏幕比例匹配的分辨率（比如 16:9、18:9、20:9），否则可能出现显示异常（如黑边、拉伸）
- **临时生效**：通过 `wm size` 修改的分辨率仅在当前会话生效，重启设备后会恢复默认
- **权限问题**：如果提示 `Permission denied`，说明设备未 root，部分厂商限制了非 root 状态下修改分辨率
- **异常恢复**：若修改后显示异常（如黑屏），无需慌张，重启设备即可恢复原始分辨率

### 总结
1. **核心命令**：修改分辨率用 `adb shell wm size 宽度x高度`，恢复用 `adb shell wm size reset`
2. **配套操作**：修改分辨率后建议同步调整屏幕密度（`wm density`），保证显示效果正常
3. **权限要求**：部分设备需要 root 权限才能修改，非 root 设备可能操作失败



# adb shell wm density 480 比adb shell wm density 240 的圖案大, 這正常嗎?

你观察到的 `adb shell wm density 480` 比 `240` 显示的图案更大，这个现象**完全正常**，而且这正是屏幕密度（DPI）调整的核心逻辑。

### 为什么密度越高图案越大？
首先要理解 **屏幕密度（DPI/Dots Per Inch）** 的本质：它代表“每英寸屏幕上的像素点数”，系统会根据这个数值来缩放界面元素（图标、文字、按钮等），核心规律是：
- **DPI 数值越大** → 系统认为屏幕像素越“密” → 会把界面元素放大显示，让图案/文字看起来更大、更清晰；
- **DPI 数值越小** → 系统认为屏幕像素越“疏” → 界面元素会缩小显示，能在屏幕上放下更多内容。

举个直观的例子（假设设备物理分辨率是 1080x2400）：
| 密度值 | 显示效果           | 实际体验                 |
| ------ | ------------------ | ------------------------ |
| 240    | 图标/文字/图案偏小 | 屏幕能显示更多内容       |
| 480    | 图标/文字/图案偏大 | 内容更清晰，但显示数量少 |

### 补充验证：反向操作更易理解
你可以实际测试一下，会更直观：
```bash
# 先设置低密度，看图案变小
adb shell wm density 240
# 再设置高密度，看图案变大
adb shell wm density 480
# 恢复默认密度（避免影响使用）
adb shell wm density reset
```

### 底层逻辑（新手易懂版）
系统渲染界面时，会先按“逻辑像素”计算元素大小，再根据 DPI 转换成“物理像素”：
- 比如一个图标设计为“20x20 逻辑像素”：
  - 240 DPI 下 → 实际占用 `20*(240/160) = 30x30` 物理像素；
  - 480 DPI 下 → 实际占用 `20*(480/160) = 60x60` 物理像素；
  物理像素翻倍，图案自然看起来大了一倍。

（注：160 是安卓的基准密度，所有密度的缩放都以它为参照）

### 总结
1. **核心结论**：480 DPI 比 240 DPI 图案更大是**正常现象**，符合安卓系统的密度缩放规则；
2. **关键逻辑**：DPI 数值与界面元素大小成**正比**（数值越大，元素越大）；
3. **实用建议**：修改分辨率后，密度值建议匹配设备原始比例（比如 1080P 设备对应 420/480 DPI，720P 对应 240/320 DPI），避免显示比例失调。
