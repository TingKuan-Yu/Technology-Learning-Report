# dockerMapping

在这个 Azure Pipeline 构建配置文件中,`dockerMapping` 用于定义 **Docker 容器内外的目录映射关系**。

## 作用

当构建任务在 Docker 容器中执行时,`dockerMapping` 将宿主机的目录挂载到容器内部,使容器能够访问宿主机上的文件。

## 在此文件中的用法

这表示:

- **宿主机路径**: `/d/mirror`
- **容器内路径**: `/d/mirror`
- 两者路径相同,实现了目录的一对一映射

## 应用场景

在这个构建配置中,`/d/mirror` 可能是:

- **镜像缓存目录**: 存储 Android 源码或依赖包的镜像
- **共享存储**: 在多个构建任务间共享数据,避免重复下载
- **持久化存储**: 保存构建过程中的中间文件或缓存

两个构建任务(Vendor_User 和 Vendor_UserDebug)都使用了相同的映射配置,确保它们能访问相同的镜像资源。
